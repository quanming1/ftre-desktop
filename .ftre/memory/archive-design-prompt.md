# 归档模块设计提示词

> 本文档是给 UI/UX 设计 Agent 的完整功能与数据描述。不包含任何 UI/UX 指导，只描述"是什么"和"能做什么"。

---

## 一、归档是什么

归档（Archive）是 AI 对一段对话的压缩摘要。当用户与 AI 的对话轮次过多时，系统会自动将历史对话压缩成一份归档，保留核心信息，释放上下文窗口。用户也可以在后续对话中引用归档，让 AI 回忆之前的上下文。

归档属于某个工作区（workspace），一个工作区下可以有任意多份归档。

---

## 二、归档文件夹是什么

归档文件夹（Archive Folder）是归档的分类容器，类似书签文件夹，不是物理磁盘目录。用于将大量归档按业务方向分组管理。

关键规则：

- 文件夹只有一层，不支持嵌套
- 一个归档可以同时属于多个文件夹（多对多关系，类似标签语义）
- 一个归档也可以不属于任何文件夹（未分类状态）
- 每个工作区的文件夹互相独立
- 同一工作区下文件夹名称不能重复
- 归档来源有两种：用户手动分类、AI 自动归类（两者结果完全一样，都是关联关系）

## 三、数据模型

### 3.1 归档（ArchiveEntry）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 唯一标识 |
| `type` | string | 归档类型，目前为 `"compaction"`（压缩归档） |
| `summary` | string | AI 生成的摘要文本，描述这段对话做了什么 |
| `content` | string | 归档的详细内容，是对话的压缩全文 |
| `meta.session_id` | string | 产生这份归档的会话 ID |
| `meta.parent_id` | string | 父归档 ID（归档可以链式衍生） |
| `meta.turn_count` | number | 压缩前的对话轮次数 |
| `meta.total_messages` | number | 压缩前的消息总条数 |
| `meta.compressed_at` | number | 压缩发生的时间戳（秒） |
| `meta.label` | string \| undefined | 用户自定义标签，可选，用于快速识别 |
| `meta.updated_at` | number \| null | 最后编辑时间，null 表示从未编辑 |
| `created_at` | number | 创建时间（Unix 秒级时间戳） |
| `folder_ids` | string[] | 该归档所属的文件夹 ID 列表，空数组表示未分类 |

### 3.2 归档文件夹（ArchiveFolder）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 唯一标识 |
| `workspace` | string | 所属工作区路径 |
| `name` | string | 文件夹名称，同工作区下唯一。如"Compaction 归档系统""消息管道优化" |
| `description` | string | 文件夹描述，可为空字符串。补充说明文件夹用途 |
| `sort_order` | number | 排序权重，越小越靠前 |
| `meta` | object | 扩展元数据，当前为空对象 |
| `archive_count` | number | 该文件夹下的归档数量（只读，服务端统计） |
| `created_at` | number | 创建时间 |
| `updated_at` | number | 最后更新时间 |

### 3.3 归档引用（ArchiveRef）—— 在聊天输入框中引用归档时的数据

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 归档 ID |
| `summary` | string | 摘要 |
| `turnCount` | number | 对话轮次 |
| `totalMessages` | number | 消息总数 |
| `label` | string \| undefined | 标签 |
| `createdAt` | number | 创建时间 |

## 四、全部用户操作

### 4.1 归档操作

| 操作 | 说明 |
|------|------|
| **查看归档列表** | 加载当前工作区的所有归档和所有文件夹，按文件夹分组展示 + 未分类归档单独展示 |
| **查看归档详情** | 展开一个归档，查看完整的 content 文本 |
| **编辑摘要** | 修改归档的 summary 字段 |
| **编辑内容** | 修改归档的 content 字段 |
| **编辑标签** | 修改归档的 meta.label 字段 |
| **引用到输入框** | 将归档作为一个引用 chip 插入到聊天输入框中，AI 可以据此回忆上下文 |
| **删除归档** | 永久删除归档，所有文件夹中的关联自动清除（不可逆操作） |
| **添加到文件夹** | 将归档加入某个文件夹（可选多个文件夹） |
| **从文件夹移出** | 将归档从某个文件夹中移出（归档本身不删除，只是不再出现在该文件夹下） |

### 4.2 文件夹操作

| 操作 | 说明 |
|------|------|
| **创建文件夹** | 指定名称和描述（可选） |
| **编辑文件夹** | 修改名称和描述（名称不能和同工作区已有文件夹重复） |
| **删除文件夹** | 删除文件夹本身和所有关联关系，文件夹内的归档不会被删除（回到未分类状态） |
| **调整排序** | 修改 sort_order 字段控制文件夹显示顺序 |
| **展开/折叠** | 展开文件夹查看内部归档，折叠隐藏 |

### 4.3 分类操作（归档 ↔ 文件夹）

| 操作 | 说明 |
|------|------|
| **拖拽分类** | 拖拽一个归档项到某个文件夹上松开，归档加入该文件夹 |
| **右键菜单分类** | 右键归档 → "添加到「文件夹名」"，或 "从「文件夹名」移出" |
| **重复加入** | 同一归档重复加入同一文件夹是幂等的，不报错 |
| **跨文件夹** | 一个归档可以同时存在于多个文件夹中（例如一个跨模块的改动同时归入「消息系统」和「Tool 系统」） |

## 五、业务约束

1. **工作区隔离**：切换工作区后，归档列表和文件夹列表完全替换为目标工作区的数据
2. **名称唯一**：同一工作区下文件夹名称不能重复，创建或重命名时如果冲突会返回 `folder_name_conflict` 错误和已存在文件夹的 ID
3. **多对多关系**：一个归档可属于 0~N 个文件夹，一个文件夹可包含 0~N 个归档
4. **AI 自动归类**：每次 AI 生成新归档时，会自动判断归入已有文件夹还是创建新文件夹，用户无需干预
5. **级联清理**：删除归档 → 自动清除所有文件夹关联；删除文件夹 → 自动清除所有归档关联，但归档本身不删
6. **归档不可恢复**：删除归档是永久操作
7. **文件夹不嵌套**：只有一层，没有子文件夹

## 六、数据量预期

| 维度 | 典型值 | 极端值 |
|------|--------|--------|
| 单工作区归档数 | 5~30 | 100+ |
| 单工作区文件夹数 | 2~8 | 20+ |
| 单文件夹归档数 | 2~10 | 50+ |
| 单归档所属文件夹数 | 0~2 | 5+ |
| summary 长度 | 20~80 字 | 200 字 |
| content 长度 | 200~2000 字 | 10000+ 字 |
| label 长度 | 2~10 字 | 30 字 |
| description 长度 | 0~50 字 | 200 字 |

## 七、上下文环境

- 这是一个桌面端 Electron 应用的侧边栏面板
- 归档列表面板与文件树、会话列表等面板平级，通过 tab 切换
- 侧边栏宽度可调整，典型宽度 260~400px
- 应用使用深色主题
- 技术栈：React 19 + Tailwind CSS 4 + Zustand + @dnd-kit（拖拽库已集成） + framer-motion（动画） + lucide-react（图标）
- 右键菜单系统已有（ContextMenu 组件），支持图标、分隔线、禁用态，不支持子菜单/嵌套
- 归档引用插入聊天框的机制已有（CustomEvent 派发）
